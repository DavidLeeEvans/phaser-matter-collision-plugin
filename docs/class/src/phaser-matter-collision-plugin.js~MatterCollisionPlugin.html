    public addOnCollideStart(options: object): function source

    Add a listener for collidestart events between objectA and objectB. The collidestart event is fired by Matter when two bodies start colliding within a tick of the engine. If objectB is omitted, any collisions with objectA will be passed along to the listener. See paircollisionstart for information on callback parameters.

    Params:

    NameTypeAttributeDescription
    options object
    options.objectA PhysicsObject | ObjectWithBody

    The first object to watch for in colliding pairs.
    options.objectB PhysicsObject | ObjectWithBody
    • optional

    Optional, the second object to watch for in colliding pairs. If not defined, all collisions with objectA will trigger the callback
    options.callback function

    The function to be invoked on collision
    options.context any
    • optional

    The context to apply when invoking the callback.

    Return:

    function

    A function that can be invoked to unsubscribe the listener that was just added.

    public removeOnCollideStart(options: object) source

    Remove any listeners that were added with addOnCollideStart. If objectB, callback or context parameters are omitted, any listener matching the remaining parameters will be removed. E.g. if you only specify objectA and objectB, all listeners with objectA & objectB will be removed regardless of the callback or context.

    Params:

    NameTypeAttributeDescription
    options object
    options.objectA PhysicsObject | ObjectWithBody

    The first object to watch for in colliding pairs.
    options.objectB PhysicsObject | ObjectWithBody
    • optional

    the second object to watch for in colliding pairs.
    options.callback function
    • optional

    The function to be invoked on collision
    options.context any
    • optional

    The context to apply when invoking the callback.
